# 幫戰覆盤新頁規格

## 1. 目標

- 獨立做一個新的 `html`
- 用來處理：
  - 單場幫戰覆盤
  - 多場累積分析
  - 玩家長期表現
  - 同職業逐人數據表
- 分析邏輯以 [幫戰覆盤分析規則.md](幫戰覆盤分析規則.md) 為準

## 2. 檔案建議

- 新檔名：
  - `battle_review.html`
- 不和 `calculator.html` 混在一起
- 資料儲存獨立，不污染既有配裝資料
- 手機版另做獨立檔：
  - `battle_review_mobile.html`

### 手機版部署資訊

- GitHub Pages repo：
  - `https://github.com/justO-arch/jol-battle-review`
- 手機網址：
  - `https://justo-arch.github.io/jol-battle-review/`
- GitHub Pages 入口檔：
  - `github_pages_mobile/index.html`

手機端建議使用方式：

- 用 `Safari` 開 GitHub Pages 網址
- 再使用：
  - `批量匯入目前所有資料`
- 匯入檔案需使用：
  - `battle_review_export_*.json`
- 不可直接使用：
  - `jol_backup_*.json`

手機版目前的實務結論：

- 不建議直接在手機上打開本地 `html`
  - 尤其 `iPhone` 的本地檔案預覽模式，常沒有真正網址，`file input` 也容易失效
- 正式使用方式改為：
  - 先部署 GitHub Pages
  - 手機以瀏覽器開網址
  - 再匯入 `battle_review_export_*.json`
- 手機版維持獨立檔：
  - `battle_review_mobile.html`
  - 不直接動桌機版 `battle_review.html`

## 3. 核心資料來源

- 單場戰報 `CSV`
- 每場的分工資料 `CSV`
  - 先由外部整理好，再匯入網站
  - 網站本身不判讀分工圖片
  - 網站只負責：
    - 匯入戰報
    - 匯入分工
    - 依 `name` 對應後做覆盤

## 4. 主要頁面區塊

新頁結構改成：

- 左側：`操作流程 + 匯入區`
- 右側：分頁區

右側分成 6 個 tab：

1. `分工`
2. `單場覆盤`
3. `玩家查詢`
4. `職業表現`
5. `對手重點人物`
6. `判讀說明`

## 5. 場次管理區

用途：

- 匯入與管理多場戰報

功能：

- 匯入單場 `CSV`
- 設定：
  - 日期
  - 對手
  - 己方名稱
  - 備註
- 顯示已載入場次清單
- 可刪除單場紀錄
- 可編輯場次資訊

第一版建議：

- 先手動管理場次
- 不做太複雜的批次匯入

## 6. 分工設定區

每場戰報需綁定分工。

每位玩家至少要有：

- `group_type`
  - `attack`
  - `defense`
  - `bodyguard`
  - `unknown`
- `subgroup`
  - 例如：
    - `進攻一隊`
    - `防守三隊`
- `special_role`
  - 可複選或文字標記：
    - `commander`
    - `healer`
    - `burner`
    - `special_task`

第一版建議：

- 以 `分工 CSV` 自動套用為主
- 表格仍保留手動微調能力
- 之後再做分工模板複用

## 7. 單場覆盤區

用途：

- 依現行規則輸出單場檢討結果

區塊內容：

### 7.1 團隊摘要

顯示：

- 總擊敗
- 總助攻
- 總資源
- 總對玩家傷害
- 總對建築傷害
- 總治療
- 總承傷

若有雙方資料，還可顯示：

- 己方 vs 對手差值

### 7.2 第一輪門檻篩選

依最新規則分成大母體主指標。

第一版不再把所有第一輪名單集中堆成一大塊表；單場覆盤上方只顯示：

- 己方整體數據
- 對手整體數據（若同份戰報可取得）

真正需要看的名單，直接以下方三欄為主：

- `優先找幹部覆盤`
- `歡迎找幹部覆盤`
- `系統難以判定`

- `進攻團 DPS（排除素問）`
  - `塔傷後 30%`
- `防守 / 保鑣團 DPS（排除素問）`
  - `人傷後 30%`
  - `擊殺後 30%`
- `進攻團素問`
  - `化羽後 30%`
  - `淨奶量後 30%`
- `防守 / 保鑣團素問`
  - `化羽後 30%`
  - `淨奶量後 30%`
- `進攻團`
  - `重傷前 30%`
- `防守 / 保鑣團`
  - `重傷前 30%`

第二輪再補看次級指標：

- 進攻團 DPS：
  - `人傷`
  - `助攻`
- 防守 / 保鑣團 DPS：
  - `助攻`
- 素問：
  - `治療量`
  - `承受傷害`
- 九靈：
  - `焚骨`
- 鐵衣：
  - 若標記坦位，再看 `承傷 + 重傷`

### 7.3 正式名單

分三欄：

- `優先找幹部覆盤`
- `歡迎找幹部覆盤`
- `系統難以判定`

每筆至少顯示：

- 名字
- 職業
- 分工
- 命中哪些規則
- 具體數據摘要

另保留：

- `人工觀察 / 備註`
  - 為單場手動文字區
  - 可輸入並保存
  - 不和系統自動名單混用

## 8. 玩家查詢

用途：

- 查單一玩家在多場中的累積表現
- 不再用大總表方式展示全員
- 目前已升級為：
  - `玩家平均值`
  - `母體中位數`
  - `母體平均數`
  - `排名 / PR / 判讀`
  同時並列的查詢頁

功能：

- 搜尋玩家名稱
- 以 checkbox 勾選要納入統計的場次
- 可切換：
  - `團別篩選`
  - `比較母體`
- 顯示：
  - 勝率
  - 優先找幹部覆盤次數
  - 歡迎找幹部覆盤次數
  - 平均擊敗
  - 平均助攻
  - 平均重傷
  - 擊殺/死亡
  - 平均人傷
  - 平均塔傷
  - 平均承傷
  - 承傷/重傷
  - 平均治療
  - 平均淨奶
  - 平均化羽
  - 平均焚骨
  - 最近一場摘要

### 團別篩選

- `全部`
- `進攻`
- `防守`
- `保鑣`
- `保鑣+防守`
- `保鑣+進攻`

口徑：

- `全部` 代表跨團別混算
- 團別篩選同時限制：
  - 玩家自己的歷史樣本
  - 比較母體

### 比較母體

- `所有職業`
- `同職業`
- `所有DPS1（奶媽除外）`
- `所有DPS2（奶媽 / 鐵衣除外）`

口徑：

- 比較不是直接拿單場原始列混算
- 而是先算每位玩家在目前篩選條件下的：
  - `多場平均`
- 再用這批玩家平均去算：
  - `母體中位數`
  - `母體平均數`
  - `排名`
  - `PR`

### 玩家摘要卡

目前摘要卡固定顯示：

- 勝率
- 分工場次
  - 顯示格式為：
    - `進攻 x 場 / 防守 x 場 / 保鑣 x 場`
- 比較條件
- 母體玩家數
- 優先找幹部覆盤次數
- 歡迎找幹部覆盤次數

不再顯示：

- `系統難以判定` 次數

### 重點對比卡

- 會依職業與主要分工，挑出最值得看的幾個主指標
- 每張卡顯示：
  - 玩家值
  - 母體中位數
  - 母體平均數
  - `PR`
  - `排名`

### 完整對比表

完整對比表目前欄位為：

- `指標`
- `玩家`
- `母體中位數`
- `母體平均`
- `母體極值`
- `標準差`
- `排名`
- `PR`
- `判讀`

補充：

- `母體極值`
  - 一般指標顯示最大值
  - `重傷` 這種低越好的指標顯示最小值
- `PR` 目前統一為：
  - `大致上越高越好`
- `判讀` 採五段制：
  - `前段`
  - `中前段`
  - `中段`
  - `中後段`
  - `後段`

### 適用性規則

數值本身仍正常顯示，但 `排名 / PR / 判讀` 會依職業決定是否適用。

- `治療 / 淨奶 / 化羽`
  - 只對 `素問` 適用
- `焚骨`
  - 只對 `九靈` 適用
- `擊敗 / 擊殺相關 / 人傷 / 塔傷`
  - 對 `素問` 顯示：
    - `- / 不適用`
- `助攻`
  - `素問` 仍適用

另外：

- 若整個比較母體在某指標都沒有有效值
  - 例如全是 `0`
  - 則該欄顯示：
    - `-`
    - `不適用`

### 建議覆盤場次統計

- 原本的「勾選場次統計」已改名為：
  - `建議覆盤場次統計`
- 目前只保留兩塊統計：
  - `優先找幹部覆盤`
  - `歡迎找幹部覆盤`

### 勾選場次統計

- 在玩家查詢頁最下方顯示三塊統計：
  - `系統難以判定`
  - `優先找幹部覆盤`
  - `歡迎找幹部覆盤`
- 取目前勾選場次中比例最高前 5 名
- 若同分則全部列入
- 顯示格式：
  - `66%：2/3`

### 能力圖

玩家查詢頁已新增能力圖對比區。

目前設計：

- 一般職業 / 鐵衣：
  - 六邊形
- `九靈`
  - 六邊形
  - 以 `焚骨` 取代 `承傷`
- `素問`
  - 治療向六邊形

目前同時顯示三張圖：

- `相對位置（PR）`
  - `玩家能力圖`
  - `母體中位數圖（n人）`
  - `母體平均數圖（n人）`
- `絕對數值`
  - `玩家絕對數值圖`
  - `母體中位數圖（n人）`
  - `母體平均數圖（n人）`

軸設計：

- 一般職業 / 鐵衣：
  - `擊敗`
  - `助攻`
  - `人傷`
  - `塔傷`
  - `承傷`
  - `生存`
- 九靈：
  - `擊敗`
  - `助攻`
  - `人傷`
  - `塔傷`
  - `焚骨`
  - `生存`
- 素問：
  - `治療`
  - `淨奶`
  - `化羽`
  - `助攻`
  - `承傷`
  - `生存`

重要口徑：

- 目前改為：
  - `PR圖`
  - `絕對數值圖`
  兩種併陳
- 目的：
  - `PR圖` 看相對位置
  - `絕對數值圖` 看離母體高標還有多遠

`PR圖` 口徑：

- 直接使用玩家查詢頁既有 `PR`
- 但保留最外圈空白
- `PR100` 不會直接滿格
- 目前約畫到：
  - `95%` 半徑

`絕對數值圖` 口徑：

- 每一軸上限定義為：
  - `max(母體極值 × 1.15, 母體平均 + 2.5σ)`

其中：

- `σ = 該指標在目前比較母體下的標準差`

`生存` 軸的處理方式：

- 不論 `PR圖` 或 `絕對數值圖`
- 都不直接把死亡數當成「越高越好」
- 一律改為：
  - `死亡越少越外圈`
  - `死亡越多越內圈`

目前結論：

- 表格用 `PR / 排名 / 判讀` 看相對位置
- 能力圖同時提供：
  - `PR圖`
  - `絕對數值圖`
- 兩者用途不同，並列顯示

### 手機版排查紀錄

- `玩家查詢` 搜尋框曾出現：
  - 桌機模擬正常
  - 真手機直向不能正常輸入
  - 真手機橫向又恢復正常
- 這代表問題不能先簡化成輸入事件本身，而要優先懷疑：
  - 手機直向排版擠壓
  - 局部區塊遮擋
  - 單欄 grid 下殘留桌機 span 樣式
- 實際排查思路定案為：
  - 先拿同頁「正常的類似區塊」對比
    - 例如 `職業表現` 搜尋區
  - 再縮小修正範圍，只動異常區塊，不大範圍重排整頁
  - 真機優先，不只看桌機 DevTools 模擬

### 手機版能力圖排版結論

- 手機版能力圖曾出現：
  - 圖偏右
  - 左邊留白過大
  - 直向時視線先看到空白
- 最後定案是：
  - 標題左對齊
  - 說明左對齊
  - 圖本體左對齊
  - 圖直接放在左側文字下方
- 這裡修的是 `svg` 與卡片內容位置，不是把整張卡片一起置中或一起右移

### 別名合併

- 只影響累積統計
- 不改單場原始名字
- 使用者可手動指定：
  - `原始名字 -> 統一名字`
- 例如：
  - `小王 -> 小丁`
- 之後多場累積時：
  - `小王`
  - `小丁`
  會統計到同一個人名下
- 別名規則保存於本頁 `localStorage`
- 下次打開頁面仍會自動套用

## 13. 規則引擎

必須直接依 [幫戰覆盤分析規則.md](幫戰覆盤分析規則.md) 實作。

核心流程：

1. 匯入戰報
2. 匯入分工 CSV
3. 用 `name` 綁定分工
4. 依分工決定主母體與特殊母體
5. 跑第一輪門檻篩選
6. 統計交集
7. 再結合特殊職責做人工判讀輔助

補充規則：

- `素問`
  - 不看塔傷 / 人傷 / 擊殺
  - 主看：
    - 化羽
    - 淨奶量
  - 次看：
    - 治療量
    - 承受傷害
- `淨奶量 = 治療 - 自己承傷`
- `九靈`
  - 額外看焚骨
- `重傷`
  - 只當警戒，不會單獨判斷成有問題
  - 並且分開看：
    - `進攻團重傷前 30%`
    - `防守 / 保鑣團重傷前 30%`
- `進攻 / 防守 / 保鑣 DPS`
  - 若 `擊殺 >= 10` 且 `擊殺/重傷 >= 2`
  - 或 `擊殺 >= 20` 且 `擊殺/重傷 >= 1.5`
  - 直接排除於自動名單之外

## 18. 覆盤資料匯出 / 匯入

用途：

- 備份目前整個覆盤頁狀態
- 換電腦 / 傳給別人時可完整還原
- 匯入後仍可繼續走一般流程追加新場次

UI 位置：

- 放在頁面最上方右上角
- 兩個按鈕：
  - `批量匯出目前所有資料`
  - `批量匯入目前所有資料`
- 固定提醒：
  - `匯入覆盤資料會覆蓋現有資料`

第一版匯出內容：

- `app`
- `version`
- `exported_at`
- `state`
  - `battles`
  - `activeBattleId`
  - `importDraft`
  - `aliasMap`
  - `settings`
- `ui_state`
  - 目前分頁
  - 玩家查詢搜尋條件
  - 職業表現篩選 / 排序條件
  - 兩個頁面的勾選場次狀態

第一版不存：

- 即時計算結果
  - 例如：
    - `優先找幹部覆盤`
    - `歡迎找幹部覆盤`
    - `系統難以判定`
- 這些都在匯入後依原始資料重新計算

第一版匯入規則：

- 只做 `覆蓋模式`
- 匯入前提示：
  - `匯入覆盤資料會覆蓋現有資料，是否繼續？`
- 匯入成功後：
  - 直接覆蓋本頁 localStorage 狀態
  - 清空目前暫存中的戰報 / 分工草稿
  - 重新渲染頁面
  - 套回 `ui_state`

## 一句話總結

這個新頁的核心不是漂亮的戰報展示，而是：

- 把現在已定的覆盤規則做成可重複執行的工具
- 支援多場累積
- 能看全員、看同職、看個人歷史
- 最後把單場印象，變成長期表現判讀
